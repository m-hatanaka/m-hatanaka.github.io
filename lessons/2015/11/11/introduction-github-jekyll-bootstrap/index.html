GitHubで技術情報を発信する

やったこと

  1. Jekyllで静的にブログ記事を執筆する環境を構築する
  2. GitHub Pagesを構築する

環境

端末：Mac (OSX Yosemite 10.10.5)

jekyll bootstrap環境構築

参考URL http://krakenbeal.blogspot.jp/2012/05/ruby-jekyll-jekyll-bootstrap.html

＜ハマった点１＞

gem経由でjekyllのインストールができない

macで環境構築する場合はxcode commandline toolをインストールしないとjekyllのインストールに失敗する模様。

http://qiita.com/tanakahisateru/items/c0eaa25dbde669b282cbに従ってコマンドラインツールのインストールを行うと、jekyllのインストールが可能となった

＜ハマった点２＞

jekyll bootstrapをクローンしたが、ローカルサーバの起動に失敗する

boostrapは最新版のjekyll 3.0に対応していない模様。

一度jekyllをアンインストールし、gem経由で2.5.3を再インストールすることで起動に成功。

github pagesを作る。

https://pages.github.comに記載の通り実施して公開用のユーザリポジトリを作成する

jekyll bootstrap上で生成した記事（_siteフォルダ）の内容を リポジトリにPushする

所感

  • github pagesは静的ページなので、コメント機能等の追加は3rdPartyのクラウドサービスとの連携が必要
  • ローカルサーバでの記事確認後、githubへのPushまでを一連の操作でできるレビューシステムを作ると便利そう
